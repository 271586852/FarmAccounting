# 资源大小优化方案

## 问题分析

### 当前情况
- **实际使用的资源**: `miniprogram/image/` 下的 6 个文件，约 24KB ✅
- **示例代码资源**: `packageAPI/`, `packageCloud/`, `packageComponent/`, `packageExtend/`, `page/` 等目录中的图片，约 2.05MB ⚠️
- **总大小**: 约 2.05MB，**超过 200KB 限制**

### 实际使用的图片文件
1. `image/calendar.png` - 日历图标
2. `image/delete.png` - 删除图标
3. `image/record.png` - 记账图标（未选中）
4. `image/record-active.png` - 记账图标（选中）
5. `image/statistics.png` - 统计图标（未选中）
6. `image/statistics-active.png` - 统计图标（选中）

## 解决方案

### 方案一：配置打包排除项（推荐）

在 `project.config.json` 中配置 `packOptions.ignore`，排除示例代码目录：

```json
{
  "packOptions": {
    "ignore": [
      "packageAPI/**/*",
      "packageCloud/**/*",
      "packageComponent/**/*",
      "packageExtend/**/*",
      "page/**/*",
      "common/**/*",
      "node_modules/**/*",
      "miniprogram_npm/**/*",
      "workers/**/*",
      "*.md",
      "*.theme.json",
      "app-darkmode.json",
      "demo.theme.json",
      "iconfont/*.css"
    ]
  }
}
```

**优点**：
- 保留示例代码，方便参考
- 不影响主包大小
- 可随时恢复

**缺点**：
- 示例代码仍占用磁盘空间

### 方案二：删除示例代码目录（彻底）

直接删除不需要的示例代码目录：

```bash
# 删除示例代码目录
rm -rf miniprogram/packageAPI
rm -rf miniprogram/packageCloud
rm -rf miniprogram/packageComponent
rm -rf miniprogram/packageExtend
rm -rf miniprogram/page
rm -rf miniprogram/common
rm -rf miniprogram/workers
```

**优点**：
- 彻底解决问题
- 减少磁盘占用
- 代码更简洁

**缺点**：
- 无法参考示例代码
- 需要确认不再使用

### 方案三：使用分包（如果示例代码需要保留）

将示例代码配置为分包，不占用主包大小。

## 推荐操作

**建议使用方案一**，配置 `packOptions.ignore` 排除示例代码目录。

这样既解决了主包大小问题，又保留了示例代码供参考。

## 优化后预期结果

- **主包图片资源**: 约 24KB（6个文件）
- **符合要求**: ✅ 远小于 200KB 限制
- **主包大小**: 预计小于 1.5MB
